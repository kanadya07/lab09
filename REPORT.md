# Отчет по лабораторной работе №9

## Тема работы

Изучение GitHub Release.

## Цель работы

Познакомиться с созданием релизов на GitHub и загрузкой собранных артефактов.

## Ход работы

### 1. Подготовка проекта

Сначала я использовала проект из лабораторной работы №8 и на его основе
создала отдельный репозиторий `lab09`.

Команды:

```bash
git clone https://github.com/kanadya07/lab08.git lab09
cd lab09
git remote remove origin
git remote add origin https://github.com/kanadya07/lab09.git
```

### 2. Проверка настроек пакетирования

В проекте уже был файл `CPackConfig.cmake`, который используется для создания
архива проекта. Для этой лабораторной работы имя пакета было обновлено на
`lab09-solver`.

### 3. Сборка пакета

Для создания архива можно использовать команды:

```bash
cmake -S . -B _build -DCPACK_GENERATOR="TGZ" -DBUILD_TESTS=OFF
cmake --build _build --target package
```

После выполнения команды в папке `_build` появляется архив, который можно
загрузить в GitHub Release.

### 4. Создание тега

Для релиза используется тег `v0.1.0.0`.

```bash
git tag v0.1.0.0
git push origin master --tags
```

### 5. Создание GitHub Release

После отправки тега на GitHub создается релиз с названием `libprint`.

В релиз загружается архив, собранный через CPack.

### 6. Проверка релиза

После загрузки артефакта можно скачать архив со страницы релиза и проверить
его содержимое:

```bash
tar -ztf lab09-solver-0.1.0.0.tar.gz
```
