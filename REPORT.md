# Отчет по лабораторной работе №8

## Тема работы

Изучение Docker.

## Цель работы

Познакомиться с созданием Docker-образа для проекта и запуском приложения в
контейнере.

## Ход работы

### 1. Подготовка проекта

Сначала я использовала проект из лабораторной работы №7 и на его основе
создала отдельный репозиторий `lab08`.

Команды:

```bash
git clone https://github.com/kanadya07/lab07.git lab08
cd lab08
git remote remove origin
git remote add origin https://github.com/kanadya07/lab08.git
```

### 2. Создание Dockerfile

После этого был создан файл `Dockerfile`. В нем описана сборка образа на
основе Ubuntu.

В образ устанавливаются необходимые пакеты:

```dockerfile
RUN apt update && apt install -yy gcc g++ cmake make ca-certificates
```

Затем проект копируется внутрь контейнера, собирается через CMake и
устанавливается в папку `_install`.

### 3. Настройка логов

Для демонстрационной программы была задана переменная окружения:

```dockerfile
ENV LOG_PATH=/home/logs/log.txt
```

Папка `/home/logs` подключается как volume, чтобы файл логов можно было
сохранить на хостовой системе.

### 4. Проверка Docker-образа

Для сборки образа используется команда:

```bash
docker build -t logger .
```

Для запуска контейнера можно использовать команды:

```bash
mkdir logs
docker run -it -v "$(pwd)/logs/:/home/logs/" logger
```

После ввода строк они должны сохраняться в файле:

```bash
cat logs/log.txt
```

### 5. Настройка CI

В файл `.travis.yml` была добавлена сборка Docker-образа:

```yaml
services:
  - docker

script:
  - docker build -t logger .
```
